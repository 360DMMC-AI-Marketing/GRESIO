2. Authentication & Security

FeatureEndpointStatusNotes
Login POST /auth/login ✅ 200 OK email + password → JWT token
Refresh Token POST /auth/refresh-token ✅ 200 OK Token refresh works
Signup POST /auth/signup ✅ Working New user registration works
2FA — ❌ Not Found No two-factor authentication available
OAuth — ❌ Not Found No Google/LinkedIn OAuth login
Password Change — ❌ Not Found No password change endpoint discovered

Auth Flow

  1. User enters email + password on login page
  2. POST to /auth/login returns JWT token
  3. Token stored in localStorage (not httpOnly cookie)
  4. Token sent as Authorization: Bearer <jwt> on all requests
  5. Token refresh via POST /auth/refresh-token

Medium Issue localStorage token storage is vulnerable to XSS attacks. httpOnly cookies would be more secure.

Medium Issue No 2FA/OAuth — password-only authentication is a security concern for enterprise clients.

Final Verdict

My360Hire is a promising ATS platform with a solid foundation for SMB recruiting. The AI job description generation works well, analytics are comprehensive (9 endpoints), and the core CRUD operations for jobs, candidates, and interviews are functional. The Free tier (5 jobs, 50 candidates, 3 AI interviews) is generous enough for thorough testing. The main blockers are the AI interview 500 error (critical — validation crashing the backend) and two missing endpoints (email template by ID, notifications mark-all-read). With these issues resolved and security enhancements (2FA, OAuth, httpOnly cookies), My360Hire could be a strong competitor in the SMB ATS space. Overall maturity: BETA stage
