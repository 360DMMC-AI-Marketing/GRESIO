Overall Assessment

FreshStockAI is a React 18 SPA (Vite, TailwindCSS) backed by a Node.js API at api.freshstockai.com with Firebase Authentication. The product shows a solid foundation with clean architecture but has critical issues that must be resolved before it can be considered production-ready for high-volume kitchens.

Key Strengths

  • Solid authentication flow (Firebase magic link)
  • Clean REST API with consistent response format
  • Good data model design (organizations, locations, vendors, inventory)
  • Margin alert system with category-based targets is thoughtfully designed
  • FSMA 204 compliance checklist is comprehensive and actionable

Key Weaknesses

  • Two 500 errors (compliance score + inventory seed) — unhandled edge cases causing backend crashes
  • AI Pricing is a marquee feature with no backend — the entire pricing module is non-functional
  • Data is sparse — the account has basic onboarding but no inventory, recipes, menu items, or orders
  • "Do This Today" AI assistant is listed but has no API and no content

API Conventions

  • Most endpoints return { success: true/false, data: ... } wrapper
  • Some endpoints return direct JSON arrays (e.g., /api/storage-locations, /api/vendors)
  • Authentication via Authorization: Bearer <firebase-id-token> header
  • Uses PATCH for partial updates (not PUT or POST)
  • Pagination uses cursor-based approach (hasMore, nextCursor)
  • Organization-scoped — all data is isolated by organizationId

Data Model Highlights

  • Organizations — Top-level tenant isolation
  • Users — Linked to organizations with roles (owner)
  • Storage Locations — Typed (dry_storage, refrigerated, freezer), with temperature control support
  • Vendors — With payment terms
  • Inventory — Items linked to storage locations and vendors
  • Recipes / Menu Items — Recipes composed of inventory items, menu items from recipes
  • Margins — Category-based target margins with alert thresholds
  • Compliance — FSMA 204 checklist with per-item status tracking

Final Verdict

FreshStockAI has a solid architectural foundation with clean REST API design, thoughtful data models, and robust Firebase authentication. However, two critical 500 errors (compliance audit score and inventory seed) and a completely missing AI Pricing backend prevent the product from being market-ready. The core features that do work (settings, vendors, storage locations, margin alerts) are well-implemented and production-ready. With focused effort on the critical and high-priority issues, FreshStockAI could become a compelling solution for small to medium restaurants within 1–2 weeks of development work.
